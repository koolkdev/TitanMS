Important notes:
I am using AngelScriptEngine now. 
For npcs the files are id.ac, for reactors the files are action.ac(the action can be found in the data for each reactor)
In AngelScript, pointers are @ and not *
and you use the pointers with . and not with ->
You can't do if(npc.getSelected()) for example, because getSelected() return int, and not boolean, so do if(npc.getSelected == TRUE/ACCEPT)
use C++ strings.(don't add the namespace before)
if you get Reactor from the map, don't forget to check that it isn't NULL (if(reacotr != NULL)...)
for NPCs, the main function is
npc_main(NPC@ npc){
}
(if you want to use things from the player functions, get the player first ex. Player@ player = npc.getPlayer(); player.changeMap(0);)
if the npc is for a quest, so the main function is start_quest_main(NPC@ npc) and end_quest_main(NPC@ npc)
for reactors, the main function is:
reactor_main(Reactor@ reactor, Player@ player){
}
